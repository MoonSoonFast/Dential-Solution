# Contributing to Dential Solution

Thank you for your interest in improving this playbook!

## How to Contribute

1. **Fork** the repository and create a new branch from `main`.
2. **Make your changes** — update or add markdown files following the existing structure.
3. **Lint your markdown** locally before pushing:
   ```bash
   npm install -g markdownlint-cli
   markdownlint '**/*.md' --ignore node_modules
   ```
4. **Open a pull request** with a clear description of what you changed and why.

## Markdown Style Guide

- Use ATX-style headings (`#`, `##`, `###`).
- Wrap lines at 120 characters where possible.
- Use fenced code blocks with a language identifier.
- Keep one blank line before and after headings and lists.

## Reporting Issues

Use the GitHub Issues tab and select the appropriate template (Bug Report or Custom).

## Code of Conduct

Be respectful and constructive. All contributions are welcome.
